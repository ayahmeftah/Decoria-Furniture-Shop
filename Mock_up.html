Home Page

Header

(Repeated on all web pages)

Our website's header has a clean and elegant design with a complementary white color scheme. The logo, featuring the website's name, is positioned on the left side as two images. In the middle, you'll find the navigation links for easy browsing. A drop-down list in the second navigation link with room category links allows seamless navigation. Hovering over a link in the drop-down list underlines it and changes the background color to grey. On the right side, three icons (Wishlist, Cart, User) and a "Sign Up/Log in" link provide functionality for saving favorites, managing the cart, and accessing the registration form. Hovering over the Wishlist and Cart icons triggers a scaling effect. Navigation links are underlined on hover, and the Sign Up/Log in link changes color and background. After registration, the Sign Up/Log in link is replaced by the first name.

Guiding Users Eyes

The homepage features a visible search bar and a "Shop Now" button to guide users. The top section is divided into two parts, with an image containing the website’s slogan on the left to balance the search bar and button, and a captivating furniture GIF on the right. A bar separates the top section from the main content of the webpage.

Easy Room Navigation

Here we have three sections for each room type (Living Room, Bedroom, Kitchen) in which we designed a clickable picture for each type with a hyperlink that takes the user directly to the room’s page. Also, for each room type there are 5 categories in which we made them as light brown rectangle boxes with links inside them that are when clicked, users will be taken to the clicked category’s section. Hovering the cursor over the category boxes will show a shadow behind them and scale will change from 1.0em to 1.03em.

Registration Form

At the end of the Home page, we have created a visually appealing registration form that seamlessly integrates with the website's design. The form features a white Bold Italic title in a light brown box which is chosen for their aesthetic compatibility. Behind the form, we added a white-Paige background image. It includes five input fields (First Name, Last Name, Password, Phone Number, and Email Address) with labels acting as placeholders within the fields. When users start entering information, the labels elegantly shift to the top for improved clarity. The input fields are designed as white boxes with thin black borders to ensure legibility. For interactivity, hovering over the submit button triggers a color change to green, accompanied by white text, while the button scales up slightly. The form has validation checks, such as a minimum of three letters for the first name and an eight-digit phone number requirement. After submission, a small pop-up appears to indicate the registration's success or provide necessary steps for completion. In case of success, the first name replaces the "Sign Up/Log in" link in the header.

Footer

(Repeated on all web pages)

We have created a minimalistic footer design that contains online generated clickable social media buttons. To maintain a visually appealing aesthetic, we have opted for a brown color scheme. The footer includes optimized icons for Instagram, Twitter, and WhatsApp. A copyright signature is positioned in the right corner, providing visitors with instant access to essential legal information. As an interactive touch, hovering over the icons triggers a subtle scale transformation from 1.0 to 1.08.


Sub-topics Pages

In the subtopic’s pages, we designed an image at the top of the page under the header. Moreover, under the room’s name in the image a search bar was added with a placeholder inside it indicating that it’s used for searching products. There are five light brown and grey coloured boxes that hold the room’s product categories as hyperlinks in white text with a shadow. The boxes colours were chosen precisely so that it matches the top picture and other elements of the page. When a user hovers on these hyperlinks the text colour changes to black, and it gets underlined showing that’s a clickable link. Also, the text’s scale will get a little bigger and the shadow colour will change to WhiteSmoke. Clicking on the hyperlinks inside the boxes navigates the user to the specified product category. Scrolling down the subtopic’s page furtherly will display the products categories, in which each category will have its products as a separate section. For each room there are five categories with six products per category. Additionally, each category section has the category name in bold black colour with an icon describing it. Furthermore, each product is displayed in a box that has product’s name, image, rating, and price. The product name is displayed at the centre of the box, on the corner left there will be the rating yellow stars and the price shown at the right corner of the box. When hovering over the box itself a black shadow shows up, and the product’s image changes to an image showing how the product looks when it’s assembled and installed in the room.

Pop Up

Upon clicking on a product box, a pop-up will appear presenting the product image on the left. On the right, the product name will be displayed in black at the top, accompanied by a description underneath. A section will be dedicated to showcasing the available colours of the product, presenting the product image in different colour variations. The pop-up will also feature two buttons; a trolley button, which allows the user to add the item to their cart, and a heart button, which enables users to add the product to their wish list. Hovering over the buttons will trigger a black shadow effect and increase the scale size. By incorporating these elements, we strive to offer users a seamless and immersive browsing experience, enabling them to explore the product details without leaving the main page.

Maintainability

To ensure maintainable code, we followed key practices. We utilized proper indentation and added comments for clarity and future maintenance. HTML tags accurately reflected content purpose, while external CSS and JavaScript files were preferred over inline methods, enabling easy updates without affecting HTML structure. Meaningful naming conventions for classes and IDs aided developers in quickly finding specific styles. CSS code was organized by grouping related styles together, facilitating efficient style discovery. Additionally, we prioritized creating appropriately named reusable functions, enhancing code reusability.
